Glossary

The vocabulary, in plain English.

The words Camel and the EIP book use, defined once so the lessons can just link here.

CamelContext
The runtime that holds and runs your routes. Camel Spring Boot creates one for you and auto-discovers every route.
Route
A single flow: from(consumer) … processing steps … to(producer). You write it in a RouteBuilder.
Endpoint
A place a route connects to, written as a URI like direct:orders or jms:queue:fulfillment. A from endpoint consumes; a to endpoint produces.
Channel
The EIP name for the pipe between a sender and receiver — in Camel, an endpoint URI.
Component
The plugin behind a URI scheme (the jms: component, the file: component). Each needs its Spring Boot starter on the classpath.
Exchange
The envelope that travels through a route. It holds the current Message (and, for request-reply, the reply).
Message
exchange.getMessage() — a body (the payload) plus headers (metadata that patterns route on).
Producer / Consumer
A consumer receives messages into a route (from); a producer sends them out (to).
DSL
Camel's fluent Java routing language — choice(), split(), filter(), and friends.
Simple language
Camel's small expression language for predicates and templates, e.g. ${body.country}.
Data Format
A reusable codec plugged into marshal() / unmarshal() to convert between bytes/text and objects (e.g. JSON via Jackson).
Marshal / Unmarshal
Marshal = object → wire format; unmarshal = wire format → object.
BOM
Bill of Materials — a special POM you import so every Camel/Spring dependency gets a compatible version without you specifying one.
Starter
A Spring Boot dependency that auto-configures a component (e.g. camel-jms-starter). One per URI scheme you use.
MockEndpoint
A test-only endpoint you set expectations on (expectedMessageCount) and then verify — how the tests prove where messages went.
Redelivery
Automatically retrying a failed step a few times before giving up (and routing to a Dead Letter Channel).
EIP
Enterprise Integration Pattern — a named, reusable solution to a messaging problem, from the Hohpe & Woolf catalog.
