Setup

Running in five minutes.

You need everyday Java knowledge and a terminal. Everything else is pinned for you.

1. Prerequisites

  • JDK 17 or 21 — check with java -version. (Adoptium Temurin is a good free build.)
  • Git and a terminal.
  • Docker Desktop — only for lesson 18 (the message-broker lesson). Everything else runs in-memory.

You do not need Maven — the repo ships a wrapper (./mvnw). You do not need a Red Hat account.

2. Clone the code

terminal
git clone https://github.com/Sreenivas-Sadhu-Prabhakara/camel-eip-course.git
cd camel-eip-course

3. Run your first tested module

This compiles Camel + Spring Boot and runs the Content-Based Router's tests:

terminal
./mvnw -pl patterns/10-content-based-router test
expected output
[INFO] Tests run: 3, Failures: 0, Errors: 0, Skipped: 0
[INFO] BUILD SUCCESS

You should see BUILD SUCCESS and three passing tests. That's the whole loop: read the route, read the test, run it.

4. Red Hat build vs. plain Apache Camel

By default the build uses the Red Hat build of Apache Camel from its public Maven repository (no login). If you're behind a firewall that blocks it, add -P upstream to use plain Apache Camel from Maven Central — same code, same lessons:

terminal
# Red Hat build (default)
./mvnw verify

# Plain Apache Camel fallback
./mvnw -P upstream verify

5. See a route run live

terminal
./mvnw -pl patterns/10-content-based-router spring-boot:run
expected output
Routing order A-1001 bound for IN
Routing order A-1002 bound for DE
Routing order A-1003 bound for US

A demo feeder sends a sample order every few seconds so you can watch it get routed by country in the log.

Stuck? The most common issue is a missing component starter (ResolveEndpointFailedException) or a wrong JDK. See the troubleshooting guide.
