About

A course you can run, not just read.

Most EIP material is either a reference catalog or a wall of prose. This course is neither: every pattern is a small Maven module you clone, run, and prove with a test.

Why it exists

Enterprise Integration Patterns are the vocabulary of system-to-system messaging. They're easy to nod along to and hard to actually build. So here each pattern is taught as a solution to one concrete problem in a running online store — ShopFlow — with a diagram, an annotated route, and a passing test that is the specification.

How it's built

  • Stack: the Red Hat build of Apache Camel for Spring Boot (Camel 4.18, Spring Boot 3.5, JDK 17/21), with a plain Apache Camel fallback.
  • Every module is identical in shape: a route, a JUnit 5 test, a README with a message-flow diagram, and sample data.
  • Continuously verified: CI builds and tests all modules on JDK 17 and 21, against both the Red Hat and upstream Camel builds.
