One story, 18 patterns

You build ShopFlow — an order pipeline — one pattern at a time.

Every lesson adds one route to the same running example: an online store's order-processing layer. By the capstone you've composed eight patterns into a real pipeline: translate → filter → de-duplicate → route → pay → fan out → publish, with a dead-letter safety net.

RUNNABLE

Clone & run it

Each pattern is its own Maven module. ./mvnw spring-boot:run and watch messages move.

TESTED

The test is the spec

Every module ships JUnit 5 tests with mock endpoints that prove exactly where each message goes.

Diagram + walkthrough

Questions

Before you start.

Do I need to know Spring Boot or Camel already?
No. The first lessons are a Spring Boot on-ramp; Camel is taught from zero. You only need everyday Java.
Is it free and open source?
Yes — code under Apache-2.0, course content under CC-BY-4.0. Clone the code repository and go.
Do I need a Red Hat subscription?
No. The Red Hat build is on a public Maven repo with no login, and a plain Apache Camel fallback works from Maven Central.
