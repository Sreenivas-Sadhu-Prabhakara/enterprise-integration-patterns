Message Filter: Drop Test and Zero-Value Orders

Discard uninteresting messages so downstream never sees them. Reach for a Message Filter whenever some inputs are simply noise — heartbeats, test traffic, empty or zero-value records — that no consumer should ever process.

The idea

Think of a spam filter on your inbox: it forwards the mail worth reading and quietly bins the rest. A Message Filter does exactly that for a message stream — it forwards a message only when it satisfies a predicate and silently drops everything else. In Camel that's the filter(predicate) block: a one-armed choice() with no otherwise(). Messages that fail the predicate just reach the end of the block and stop — no branch, no trace.

The scenario

ShopFlow's order stream is polluted by two kinds of junk that must never reach fulfilment: zero-value orders (nothing to fulfil) and test-flagged orders (QA smoke traffic). We keep a message only when it is a real order worth money — totalAmount > 0 and testFlag == false — and route survivors to a single {{ep.out}} placeholder. Everything else is dropped into the void.

The route

MessageFilterRoute.java
from("direct:orders")                                         // (1) the channel orders arrive on
    .routeId("filter")                                        //     always name a route (tracing, metrics, tests)
    .log("Considering order ${body.orderId} (amount=${body.totalAmount}, test=${body.testFlag})")
    .filter(simple("${body.totalAmount} > 0 && ${body.testFlag} == false")) // (2) the Message Filter itself
        .log("Order ${body.orderId} passed the filter -> forwarding")       // (3) only survivors get here
        .to("{{ep.out}}")                                     // (4) the one destination for kept messages
    .end();                                                   // (5) messages that fail the predicate are dropped
  1. (1) the route consumes orders from an in-memory direct: channel.
  2. (2) filter(simple(...)) is the Message Filter — a single Simple predicate combining an amount test and a boolean flag test with &&.
  3. (3) only messages that satisfy the predicate flow past the filter, so this log line and the .to(...) run for survivors only.
  4. (4) the one destination is a {{ep.out}} placeholder — production points it at a real direct:/jms: endpoint while tests point it at a mock.
  5. (5) .end() closes the filter block; messages that failed the predicate simply reach here and are silently dropped.

Run it

terminal
./mvnw -pl patterns/09-message-filter spring-boot:run

A demo feeder injects a rotating sample every 3 seconds — one real order, one zero-value order, one test order — so you'll watch the real one land on the kept path while the other two are considered and then silently dropped. Behind a firewall? Add -P upstream.

The test is the spec

MessageFilterTest.java
@Test
void keepsRealOrdersAndDropsRejectableOnes() throws Exception {
    Order valid = new Order("A-1001", 999, false);   // real money, not a test  -> KEPT

    // Exactly one message should survive the filter, and it must be the valid order.
    out.expectedMessageCount(1);
    out.expectedBodiesReceived(valid);

    template.sendBody("direct:orders", valid);                         // KEPT
    template.sendBody("direct:orders", new Order("A-1002", 0, false)); // DROPPED (fails totalAmount > 0)
    template.sendBody("direct:orders", new Order("A-1003", 250, true)); // DROPPED (fails testFlag == false)

    MockEndpoint.assertIsSatisfied(context);
}

In the test profile {{ep.out}} resolves to mock:out, so the spec sends three orders — a valid one, a zero-value one, and a test-flagged one — and asserts that exactly one reaches the mock. Dropped messages leave no trace, so we prove them by their absence. Run just the tests:

terminal
./mvnw -pl patterns/09-message-filter test
Try it Silent drop is fine for noise, but wrong for anomalies. Add a second route (or an explicit else) that sends the rejected orders to a mock:invalid Invalid Message Channel instead of the void, and assert both mocks.

What you learned

  • A Message Filter forwards a message only when it satisfies a predicate and silently drops everything else.
  • filter(predicate) is a one-armed choice() — reach for it when the "else" is simply "drop it".
  • Drop noise, capture anomalies: when rejects need auditing or replay, route them to an Invalid Message Channel rather than the void.
