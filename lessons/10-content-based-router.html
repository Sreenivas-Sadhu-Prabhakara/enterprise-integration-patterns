Content-Based Router

Send each message to a different destination based on its content. Reach for it whenever "where this goes next" depends on "what's inside".

The idea

Imagine a mailroom sorting letters by ZIP code. A Content-Based Router does the same for messages: it inspects each one and forwards it down exactly one branch. In Camel that's the choice() block — a chain of when(predicate) branches with a final otherwise(). Always provide the otherwise(), or unmatched messages quietly vanish.

The scenario

ShopFlow fulfils orders differently by region: IN goes to the local warehouse, EU countries (DE, FR, ES, IT) go to the EU hub, and everything else goes to an international handler. We route each order by its country.

The route

ContentBasedRouterRoute.java
from("direct:orders")                                     // (1) the channel orders arrive on
    .routeId("content-based-router")                      //     always name a route
    .log("Routing order ${body.orderId} bound for ${body.country}")
    .choice()                                             // (2) the Content-Based Router
        .when(simple("${body.country} == 'IN'"))          // (3) domestic
            .to("{{ep.domestic}}")
        .when(simple("${body.country} regex '(DE|FR|ES|IT)'"))  // (4) EU hub
            .to("{{ep.eu}}")
        .otherwise()                                      // (5) ALWAYS handle the rest
            .to("{{ep.international}}")
    .end();
  1. (1) the route consumes orders from an in-memory direct: channel.
  2. (2) choice() opens the router; it evaluates each when in order and takes the first match.
  3. (3)(4) predicates are written in the Simple language — an equality test and a regex over the order's country.
  4. (5) otherwise() catches everything unmatched. The branch targets are {{ep.*}} placeholders so production points them at real channels while tests point them at mocks.

Run it

terminal
./mvnw -pl patterns/10-content-based-router spring-boot:run

A demo feeder sends orders from rotating countries every few seconds — watch each land on the matching branch in the log. Behind a firewall? Add -P upstream.

The test is the spec

ContentBasedRouterTest.java
@Test
void domesticOrderGoesDomestic() throws Exception {
    domestic.expectedMessageCount(1);
    eu.expectedMessageCount(0);
    international.expectedMessageCount(0);

    template.sendBody("direct:orders", new Order("A-1001", "IN", 999));

    MockEndpoint.assertIsSatisfied(context);
}

In the test profile the branch endpoints resolve to mock:domestic, mock:eu, mock:international, so each test asserts exactly one branch fires and the others stay empty. Run just the tests:

terminal
./mvnw -pl patterns/10-content-based-router test
Try it Add a fourth branch for orders over €10,000 to a mock:highvalue endpoint, and write the assertion that proves it.

What you learned

  • A Content-Based Router forwards each message down exactly one branch, chosen by its content.
  • choice().when(...).otherwise() is the Camel DSL — and otherwise() is not optional.
  • Property-placeholder endpoints let the same route be production-real and test-assertable.
